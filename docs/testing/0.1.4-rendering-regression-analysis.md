# WebMMD 0.1.4 — Rendering Regression Analysis

This document details the analysis of the rendering regressions discovered in WebMMD 0.1.3 using the Lynette PMX model (`ref/test.zip`).

## Verified Hypotheses

### 1. Disabling outlines restores colored surfaces

- **Observation:** When outlines are enabled, the entire model surface turns into a black silhouette.
- **Cause:** The outline pass runs _after_ the main model pass and writes to the depth buffer without proper front-face culling configuration. This causes the expanded inverted-hull outline geometry to completely cover and overwrite the colored surfaces of the model. Disabling outlines or drawing them _before_ the main pass solves this.

### 2. Disabling back-face culling restores surfaces

- **Observation:** Some model surfaces disappear entirely when culling is enabled, but reappear when `cullMode` is set to `"none"`.
- **Cause:** WebGPU's default winding is Counter-Clockwise (`"ccw"`). Since PMX models use Clockwise (`"cw"`) triangle winding, a default `"ccw"` pipeline with `"back"` culling incorrectly culls the actual front-facing triangles.

### 3. Changing `frontFace` changes which surfaces disappear

- **Observation:** Setting `frontFace` to `"cw"` corrects the culling, making front-facing triangles visible and culling back-facing triangles.
- **Cause:** PMX model geometry is wound Clockwise (`"cw"`). Specifying `frontFace: "cw"` aligns the WebGPU pipeline with the model's coordinate and winding conventions.

### 4. File extension-based transparency classification is incorrect

- **Observation:** Eyelashes, hair details, skin parts, and clothing using `.png` or `.tga` textures are all classified as transparent/blended.
- **Cause:** A binary file-extension check (`endsWith(".png") || endsWith(".tga")`) classifies all textures that support alpha as blended. Most of these textures do not actually contain semi-transparent pixels, meaning they should be classified as opaque or cutout.

### 5. Blended materials lack depth writing and expose opposite-side geometry

- **Observation:** Clothes and hair show see-through artifacts, revealing internal body meshes and back-facing geometry.
- **Cause:** Transparent materials disable depth writes (`depthWriteEnabled: false`). When opaque materials are misclassified as transparent, they do not write to the depth buffer, allowing subsequently rendered meshes to be drawn on top of them, regardless of distance.
