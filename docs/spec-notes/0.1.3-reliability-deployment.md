# WebMMD 0.1.3 Reliability & Deployment Spec Notes

This release hardens the WebMMD application, improves browser validation, adds robust error recovery, and configures static deployment with offline capabilities.

## 1. Reliability & Error Recovery

To ensure the viewer remains stable under real-world runtime conditions:

- **Visibility-based Frame Loop Suspension**: The application detects tab visibility changes using the browser Page Visibility API. When the page is hidden, the main animation frame request loop is suspended to conserve battery, CPU, and GPU resources. Rendering automatically resumes when the tab becomes visible.
- **Transparent WebGPU Device Lost Recovery**: In the event of a GPU driver crash or device reset (e.g. system sleep/wake), the viewer attempts transparent recovery up to 5 times. It re-initializes the WebGPU context, clears texture caches, and automatically re-uploads the active model data, textures, and runtime state, ensuring a seamless user experience.
- **Strict VFS and Path Resolution**: Texture reference path resolution behaves deterministically across platforms (supporting backslashes, mixed casing, and relative traversal) using a Virtual File System (VFS). A case-insensitive lookup fallback resolves filenames if the specific directories differ.

## 2. Safety-First Worker Threading

All heavy CPU parsing operations are delegated to background web workers to prevent UI thread freezing:

- **ZIP Extraction**: Handled via `zip.worker.ts` utilizing `fflate`. The worker enforces strict safety limits: a maximum of 2,000 files and 500 MB total unzipped size.
- **Model Parsing**: Handled via `parser.worker.ts`, translating raw PMX binary data into structured packed format for fast GPU transmission.

## 3. Static Deployment & PWA Configuration

The web application is fully prepared for static hosting and offline usage:

- **PWA Manifest (`manifest.json`)**: Configured with theme colors, metadata, and branding icons.
- **Service Worker (`sw.js`)**: Implements network-first cache fallback strategies for core application resources, WASM bundles, CSS, and fonts, allowing full offline model inspection.
- **CI/CD Pipeline**: GitHub Actions workflow (`.github/workflows/deploy.yml`) automates verification (formatting, linting, tests) and builds/deploys the production application directly to GitHub Pages.
